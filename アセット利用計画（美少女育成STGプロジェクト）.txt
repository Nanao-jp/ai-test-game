アセット利用計画（美少女育成STGプロジェクト）

プロジェクトの基盤は Game Creator 2 を中心に構築する。
全体の目的は「AI支援＋高品質アセットで、短期間に商用水準の美少女育成STGを実現すること」。
このため、各アセットは独立した役割を持ちながらも、すべてGC2のイベント・データ構造を軸に連携させる。

Game Creator 2

開発の中核。
キャラクターのステータス、レベル、装備、好感度などをScriptableObjectベースで管理し、
育成・クエスト・ホーム画面など非戦闘パートのロジックをGUIで定義する。
AIによるコード生成を最小限に抑え、全体をイベントドリブン型で運用する。
Dialogue SystemやTop Down Shooter ToolKitへのデータ受け渡しもGC2を介して行う。
プロジェクト全体の“OS”として機能する想定。

Modern UI Pack / DOTween Pro

UIと演出を担当。
Modern UI Packは、全画面共通のデザイン基盤として採用する。
ホーム、編成、育成、リザルト画面など、全シーンで統一した美少女ゲーム調のトーンを保つ。
DOTween Proは演出面での補助として、UIアニメーション・スキル発動時のズーム・リザルト演出を制御。
アズレンやブルアカのような「スムーズに動くUI」「ボタンの弾力感」「スキル発動時の一瞬の強調」など、
細かい“手触り”をここで再現する。

Dialogue System

会話・シナリオ・スキル演出を統合するシステム。
美少女キャラのセリフ、スキルカットイン、バトル開始時のボイス演出など、
ADVパートと戦闘パートの橋渡しを担う。
GC2のステータス変数やフラグと連携して、
「好感度が一定以上で特殊セリフが出る」「育成段階に応じてスキルボイスが変わる」といった分岐を制御する。
また、テスト段階では簡易会話イベントの作成に使い、
シナリオ量が増えるリリース前にはGoogle Sheets連携による外部管理へ移行する想定。

Top Down Shooter ToolKit

戦闘パートの基盤。
縦スク仕様へ改造して使用し、機体のパラメータをGC2の育成データと連動させる。
攻撃力、射撃速度、スキルゲージ回復などをGC2から受け取って反映。
敵のAI・弾幕パターン・Wave制御などは既存機能を活かしつつ、
DOTweenで演出を追加し、戦闘画面をリッチ化する。
スマホ向け最適化のため、物理処理は軽量化し、UI操作系をModern UI Packで再構築する。
カメラワークやスキル発動演出はCinemachineと組み合わせて調整する予定。

Easy Save 3（リリース前導入）

最終段階でセーブシステムをEasy Save 3に置き換える。
GC2標準セーブをテスト段階では使い、リリース準備に入った時点で
ES3へ移行して暗号化・クラウド対応を実現する。
セーブデータはステータス、装備、好感度、スキルレベル、進行クエストなどを対象とし、
複数端末での同期やチート防止を含む運用を前提に設計する。

全体方針

開発初期はGC2＋TDS-TK＋Modern UI Packでプロトタイプを構築し、
会話・演出が必要になった段階でDialogue Systemを組み込む。
全アセットが安定動作した段階でEasy Save 3を導入し、
商用配信を見据えた完全セーブ構造に移行する。
この4＋1構成で、UI・戦闘・育成・会話・保存が統一された
AI駆動型の美少女育成STG基盤を完成させる。
